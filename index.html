Q02 Describe a dificult situation you met, explain the tools to solved it and what you learnt from it

I undertook to create an educational website with a web designer friend. I wanted both to work with someone with an artistic vision and to produce a Game of Life type cellular automata. Several problems arose: a cellular automaton is computationally expensive. I had already noticed that the React framework did not allow me to achieve optimal performance when deep components are very active and update, even occasionally, the state of the entire application. I could have used a state management library but not without impact on my bundle size. After some research, I decided to switch to another framework, SolidJS, which is very efficient and very light. Instead of updating the entire application in a top-down manner in the component tree, SolidJS only updates the components that need to be updated (fine-grained reactivity) through the principle of signals. Also, SolidJS does not rely on a virtual DOM like React or Vue but on the real DOM, sparing the most expensive phase in other frameworks when rendering components, the reconciliation phase of the DOM and the virtual DOM. SolidJS is close to native JavaScript and allows for easier separation of UI and logic like hooks. This was a great advantage in terms of code organization. Both the logic and the graphical interface can be tested independently and reused in other projects. On this occasion, I also learned to translate React code to SolidJS, taking advantage of the richness of the React ecosystem. Using another framework allowed me to take a step back from my knowledge. I have considerably enriched my development work environment by using better linting practices and better project organization.
